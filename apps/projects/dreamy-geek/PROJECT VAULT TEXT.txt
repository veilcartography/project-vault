# Reseller HQ — Notion Systems + Agents

## Product / project summary

**Reseller HQ** is a Notion-based operating system for managing a UK reseller business from sourcing to sale. It brings together auction research, AI-assisted buying decisions, lot intake, invoice tracking, listing preparation, sales-channel management, platform notifications, and daily email review into one structured workflow.

The core system is built around a **Lot-first workflow**: every won auction lot becomes the source-of-truth record, then invoices and listings connect back to that lot. This prevents scattered information, duplicate cost entry, and lost product context. The workflow is documented as **Lots → Invoices → Listings** and is controlled through the **Reseller Command Center**. [[1]](https://app.notion.com/p/Reseller-Command-Center-3041fbe6474681139194c3b669dde788?pvs=21)[[2]](https://app.notion.com/p/System-Overview-Lots-Invoices-Listings-7ed836f9613f4d72a68055b36945afba?pvs=21)[[3]](https://app.notion.com/p/Reseller-Workflow-Quick-Guide-Lots-Invoice-Listings-873242293ee3490c808cd1eafdec432d?pvs=21)

Reseller HQ is not just a tracker. It is designed as a repeatable, AI-supported business system that can become a future sellable template or digital product.

## 1. Reseller Command Center

The **Reseller Command Center** is the main navigation hub for reseller operations. It acts as the front door for the whole workflow and keeps the daily process simple: **Lots → Invoice → Listings**. [[1]](https://app.notion.com/p/Reseller-Command-Center-3041fbe6474681139194c3b669dde788?pvs=21)

Its purpose is to stop reseller admin becoming scattered across emails, screenshots, invoices, marketplace tabs, auction sites, and notes. Instead, it gives one place to start and one route through the process.

### Main functions

- Holds the top-level workflow checklist.
- Links to each operational database.
- Separates new stock intake, invoice processing, and listing creation.
- Stores workflow files such as invoices, CSVs, screenshots, and source documents.
- Links to guidance pages, VA instructions, and listing assistant resources.

### Core rule

The golden rule is: **Lot-first. Links now, costs later.**

This means:

1. Create one lot row per won auction lot.
2. Capture auction and retail links first.
3. Add invoice data later when the invoice arrives.
4. Create one listing row per platform once the lot is ready to sell.

## 2. Lots → Invoices → Listings system

The main reseller workflow is a three-database system designed to keep stock, costs, and listings connected. [[2]](https://app.notion.com/p/System-Overview-Lots-Invoices-Listings-7ed836f9613f4d72a68055b36945afba?pvs=21)

### A. Lots / Master Intake

The **Lots / Master Intake** database is the source of truth for everything won at auction. Each row represents one auction lot.

It stores:

- Lot title
- Auction house
- Lot number
- Auction URL
- Retail research URLs
- Screenshots or photos
- Intake status
- Hammer price
- Final lot cost
- Item count
- Unit cost
- Fees and tax
- Related invoice
- Related listings

The system intentionally separates **links captured** from **costs confirmed** so that you do not need to fill every field at once.

### B. Financials & Invoices

The **Financials & Invoices** database stores one row per invoice batch. One invoice can link to many lots. [[2]](https://app.notion.com/p/System-Overview-Lots-Invoices-Listings-7ed836f9613f4d72a68055b36945afba?pvs=21)[[3]](https://app.notion.com/p/Reseller-Workflow-Quick-Guide-Lots-Invoice-Listings-873242293ee3490c808cd1eafdec432d?pvs=21)

It stores:

- Invoice number
- Invoice date
- Auction house
- Total amount
- Invoice PDF / screenshot
- Related lots

The invoice database is treated as a batch wrapper, not the main place for editing costs. The cost inputs should live on the Lot records to avoid duplication and confusion.

### C. Sales Tracker

The **Sales Tracker** database stores one row per platform listing. A single lot can generate multiple sales-channel rows.

Example:

- One Vinted listing
- One eBay listing
- One Facebook Marketplace listing
- One Gumtree listing

Each listing links back to the original lot, so pricing, photos, descriptions, and sales outcomes stay connected to the source stock.

The Sales Tracker can track:

- Platform
- Listing title
- Draft stage
- Status
- Price
- Listing URL
- Seller photos
- Buyer-info images
- Platform rules
- Notes
- Related lot

The **Reseller Workflow — Quick Guide** is the simple operating manual for the whole reseller system. It explains what to do at each stage and prevents over-filling databases too early. Minimum fields:

- Lot title
- Auction page URL
- Main retail URL
- Additional retail URLs
- Intake status = Links captured

Do not fill yet:

- Hammer price
- Final lot cost
- Item count

Then update each lot with:

- Hammer price
- Final lot cost

The system then calculates:

### Step 3 — Create listings

Listing rows track:

- Platform

- Photos
- Price

## 4. Auction AI System

The **Auction AI System** is the buying-decision layer of Reseller HQ. It is designed to help assess auction opportunities before bidding, so purchases are disciplined rather than emotional. [[4]](https://app.notion.com/p/Auction-AI-System-3601fbe64746804b8e16d46aaf335ebc?pvs=21)

It supports the sourcing stage before stock enters the Lots → Invoices → Listings workflow.

### Purpose

The Auction AI System helps decide whether to:

- **Bid**
- **Watch**
- **Ignore**

It uses profit rules, delivery estimates, storage constraints, and resale value to protect cash and avoid low-margin purchases.

### Daily workflow

1. Check incoming Notion auction alerts.
2. Use Highlight or browser assistance to scan deals.
3. Review promising lots.
4. Send key deals to the reseller decision agent / AI workflow.
5. Decide: bid, watch, or ignore.
6. Log useful deals and patterns.
7. Execute only on high-confidence opportunities.

### Decision rules

The system includes:

- Profit thresholds
- Delivery-cost rules
- Collection vs delivery comparisons
- Storage rules
- Energy vs value scoring
- High-confidence testing mode
- Pattern logging
- Mistake logging
- Deal-learning loops

### Product potential

This system is already structured like a sellable product:

- Notion template
- Excel / tracker companion
- AI reseller prompt pack
- Auction decision guide
- Deal-scoring dashboard

## 5. Auction Command Center — Alert Processing

The **Auction Command Center — Alert Processing** page is the live reference for handling auction alerts and outbid notifications. [[5]](https://app.notion.com/p/Auction-Command-Center-Alert-Processing-15b40f90096e4addae60c3c718554c4e?pvs=21)

It turns messy auction emails into fast **Bid / Avoid / Re-bid / Walk Away** decisions.

### What it handles

- New auction alerts
- Outbid notifications
- Ending-soon opportunities
- High-brand low-bid checks
- Auction-house logistics
- Collection vs delivery calculations
- Profit margin checks

### Alert processing logic

The system uses three main checks:

1. **Outbid alerts**  
    
    Check whether the new bid still leaves a £25–£50 profit margin.
    
2. **New alerts**  
    
    Cross-reference the item with a lookout list and calculate the max bid.
    
3. **£5 lottery check**  
    
    If a strong brand is ending soon and still near £5, flag it as a low-risk opportunity.
    

### Outbid decision matrix

- Under max bid → **Re-bid**
- At max bid → **Last call**
- Over max bid → **Walk away**

### Supporting databases

The page includes:

- **Auction Alert Log**
- **Outbid Tracker**

These are used to capture and process alert data as structured records.

## 10. Email Manager Agent

The **Email Manager** is the main active custom agent connected to Reseller HQ. It is the only daily auto-running agent in the system. [[6]](https://app.notion.com/p/Systems-Overview-Notion-Agents-Databases-736b9e6cb77d4070883c00bd45a30ad7?pvs=21)

### Trigger

- Runs daily at **09:00 Europe/London**.
- Checks the selected reseller inbox setup.
- Other accounts can be forwarded or imported into the main mailbox.

### Main output

It creates a daily page called:

**📬 Daily Email Summary — YYYY-MM-DD**

This daily summary is designed as a command brief, not just an email digest.

### What it extracts

The Email Manager looks for:

- Auction alerts
- Outbid notifications
- Buyer messages
- Marketplace notifications
- Finance and subscription emails
- Security alerts
- Replies needed
- Deadlines
- Platform activity
- Important updates

### Daily summary sections

The summary groups email into decision-friendly sections such as:

- Do Today — Top 3
- Needs Reply / Action Required
- Finance / Subscription
- Verify / Review
- Vinted and priority platforms
- Auto-archived
- Come Back Later
- Possible Junk

### Credit-saving behaviour

The agent is designed to minimise unnecessary AI use:

- Scans at thread level first.
- Deep-reads only important emails.
- Avoids deep-reading low-priority newsletters and marketing digests.

### Retention rule

Daily Email Summary pages are archived after 7 days, not deleted. This keeps the dashboard clean while preserving history.
