# Digital Workspace Hub User Guide

## What This Hub Is

`project-hub.html` is your local control panel for the web pages and tools you are preparing to publish.

It does not upload anything by itself. It gives you one clean place to open, compare, and launch the HTML pages that have been kept, cleaned, extracted, or prepared for GitHub.

## Recommended Workflow

1. Open `project-hub.html`.
2. Open each local page from the hub.
3. Confirm which page is the newest and cleanest version.
4. Open the matching file inside the clean GitHub upload package.
5. Upload that clean file to the matching GitHub repository.
6. Wait a minute or two for GitHub Pages to refresh.
7. Open the public link from the hub and check the live page.

## Important Notes

- The hub is local-first. It does not send your data anywhere.
- The clean upload package is the safest source for public files.
- The Manus exports may still depend on Manus-hosted runtime files. If those pages need to become fully independent, rebuild them as standalone HTML apps later.
- Do not delete archived old versions until the live GitHub pages have been tested.
